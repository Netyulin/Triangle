# 签名模块补充说明（2026-04-12）

## 本次补充的关键改动

### 1. 用户签名流程改为分步展示

用户前台 ` /dashboard ` 现在按以下顺序展示签名流程：

1. 下载 `mobileconfig` 描述文件获取 UDID
2. 选择当前账号下的目标设备
3. 选择签名证书与描述文件
4. 上传 IPA 并创建签名任务

这样做的目的，是避免用户在没有 UDID 的情况下提前看到不该操作的步骤，也方便多个设备时手动切换目标 UDID。

### 2. 用户自签名资产改为“成套上传”

用户侧不再鼓励把证书和描述文件分开上传理解。

现在新增了“成套上传”逻辑：

- 一次同时上传 `.p12`
- 同时填写证书密码
- 同时上传 `mobileprovision`
- 后端校验通过后，再一起保存并设为当前用户默认方案

当前用户页面会明确提示：

- 证书与描述文件是强绑定关系
- 上传时必须成套提交
- 校验失败则不会入库

### 3. 新增证书与描述文件匹配校验

后端新增校验逻辑：

- 从 `.p12` 中解析证书团队标识
- 从 `mobileprovision` 中解析 `TeamIdentifier / ApplicationIdentifierPrefix`
- 只有团队标识一致，才允许保存

当前已用现有文件验证通过：

- `push.p12`
- `pushon.mobileprovision`

匹配结果：

- 证书团队标识：`MK8M2ZWCTL`
- 描述文件团队标识：`MK8M2ZWCTL`

### 4. 新增签名产物 24 小时自动清理

用户签名成功后生成的这些临时文件：

- 已签名 IPA
- manifest.plist
- 下载链接
- 安装链接

现在默认只保留 24 小时。

后端处理方式：

- 签名完成时写入 `expiresAt`
- 服务启动时执行一次清理
- 之后每小时执行一次清理
- 超时后删除签名文件与清单文件
- 同时清空任务里的下载链接和安装链接
- 标记 `cleanedAt`

前端表现：

- 任务卡片会显示“链接保留到”
- 文件被清理后，会提示“该任务产物已超过 24 小时，文件已自动清理”

### 5. 修复下拉框显示 `????`

问题根因不是前端组件，而是数据库里两条系统签名资产的名称字段已经被写坏。

已处理：

- `sign_certificates` 中系统证书名称改回：`系统推送证书`
- `sign_profiles` 中系统描述文件名称改回：`系统推送描述文件`
- 描述文件备注改回：`默认系统描述文件`

另外，服务端现在也加了兜底：

- 如果老数据名称是纯 `????`
- 接口返回时会自动回退到更合理的显示名
