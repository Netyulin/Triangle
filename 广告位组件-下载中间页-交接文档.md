## 一、需求概述

基于 Google AdSense + CPS 联盟的商业模式，实现广告展示和下载返利功能：
1. **AdSense 代码位** - 在首页、软件详情页、下载中间页植入 Google AdSense
2. **CPS 联盟中间页** - 下载经过中间页，跳转到联盟跟踪链接并记录转化
3. **Per-mirror CPS 链接** - 每个下载镜像可独立设置 CPS 链接

## 四、数据模型（v7.0 修订版）

> **重大变更**：AdSense + CPS 模式下，原 AdSlot/AdContent/DownloadLog 模型已废弃，不再需要自建广告管理后台。

### 4.1 现有模型复用

```prisma
// App.downloadLinks 格式变更（已存在的字段，语义扩展）：
// Array<{ name: String, url: String, affiliateUrl?: String }>
// affiliateUrl: 该镜像的 CPS 联盟跟踪链接，为空则表示无 CPS 直接跳转

model App {
  // ... 已有字段 ...
  downloadLinks Json?  // [{ name, url, affiliateUrl }]
  // affiliateLink 字段（废弃，保留但不使用）
}
```

### 4.2 新增模型

```prisma
model CpsDownload {
  id           String   @id @default(cuid())
  softwareSlug String
  software     App      @relation(fields: [softwareSlug], references: [slug])
  mirrorIndex  Int      @default(0)   // downloadLinks 数组下标
  ip           String
  userAgent    String?
  clickTime    DateTime @default(now())
  status       String   @default("clicked") // clicked, converted, expired

  @@index([softwareSlug])
}
```

### 4.3 API 路由（Phase 1）

| 方法 | 路由 | 说明 |
|------|------|------|
| GET | `/api/download/[slug]` | 获取下载信息（含 affiliateLinks），不再创建 DownloadLog |
| POST | `/api/download/[slug]/cps-click` | 记录 CPS 点击（新建 CpsDownload） |
