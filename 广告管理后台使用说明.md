# 广告管理后台使用说明

适用项目目录：`E:\Project\Triangle`

最后更新：2026-04-13

## 后台页面说明

当前后台和广告相关的主要页面有 3 个：

1. `/admin/ad-slots`
2. `/admin/ad-contents`
3. `/admin/ads-stats`

它们分别负责：

- 广告位管理：定义广告位、尺寸、位置、启用状态
- 广告内容管理：管理自营广告素材
- 广告数据：查看广告位、内容与 CPS 下载相关统计

## 当前前台广告的真实工作方式

这里要特别说明清楚：

- 后台这套广告位 / 广告内容系统是项目自己的广告管理能力
- 前台首页、详情页、下载页当前主广告位实际走的是 `AdSenseSlot` 组件
- 所以“后台新增一条广告内容”不等于前台立刻就能看到新的 Google 广告

也就是说：

- Google AdSense 是否展示，最终取决于：
  - 前台广告组件是否正常
  - Google 广告脚本是否能加载
  - 终端用户网络能否访问 Google 广告域名
  - Google 是否真的返回广告素材

## 正式站广告位真验证方案

### 验证目标

判断的不是“页面上有没有空白占位”，而是：

**Google 是否真的已经开始投放广告。**

### 通过标准

只有同时满足下面几项，才算真正开始投放：

1. `adsbygoogle.js` 成功加载
2. 页面发出了后续广告请求
3. `ins.adsbygoogle` 内出现了 `iframe`
4. 页面肉眼可见广告素材
5. AdSense 后台后续能看到展示量增长

### 不算真正投放的情况

下面这些都不能算真正投放：

- 只有广告容器，没有 `iframe`
- 只有脚本标签，没有脚本成功加载
- 只有占位高度，没有广告内容
- 只有前端不报错，但没有任何 Google 广告请求

### 推荐验证环境

不要优先用当前异常网络环境判断。

推荐顺序：

1. 海外网络或能稳定访问 Google 的网络
2. 另一条正常公网网络
3. VPS 所在环境
4. 当前本地异常网络，仅作参考

### 浏览器验证步骤

打开正式站首页：

- [www.sanjiaosoft.com](https://www.sanjiaosoft.com)

然后打开开发者工具，检查 3 个面板：

- `Network`
- `Console`
- `Elements`

#### 第一步：检查广告脚本

在 `Network` 里搜索：

- `adsbygoogle.js`

正常应看到：

- 来源：`pagead2.googlesyndication.com`
- 状态：`200`

#### 第二步：检查后续广告请求

继续在 `Network` 里搜索：

- `doubleclick`
- `googlesyndication`
- `gen_204`
- `activeview`

只要出现后续广告请求，说明脚本不只是加载了，而是在真正工作。

#### 第三步：检查广告容器

在 `Elements` 里找到：

- `ins.adsbygoogle`

真正投放时，里面通常会出现：

- `iframe`

如果始终没有 `iframe`，通常还不能算真正投放。

#### 第四步：肉眼确认广告素材

页面上应能看到：

- 图片广告
- 文字广告
- 推荐卡片广告
- 或其他可点击广告内容

### 控制台辅助检查

可以在 Console 中执行：

```js
typeof window.adsbygoogle
```

正常情况下不应为 `undefined`。

再执行：

```js
Array.from(document.querySelectorAll('ins.adsbygoogle')).map(el => ({
  slot: el.getAttribute('data-ad-slot'),
  hasIframe: !!el.querySelector('iframe'),
  innerHTMLLength: el.innerHTML.length
}))
```

重点看：

- `hasIframe`
- `innerHTMLLength`

### 结果判断

#### 通过

满足下面大部分即可判断广告已经真正开始投放：

- 脚本 200
- 有后续广告请求
- 有 `iframe`
- 页面看得到广告素材

#### 待观察

如果出现：

- 脚本 200
- 有请求
- 暂时无可见广告

说明：

- 代码链路基本接通
- 但 Google 暂时还没稳定填充广告

#### 不通过

如果出现：

- 脚本加载失败
- 没有后续广告请求
- 没有 `iframe`
- 页面空白

则当前广告位还不能算真正投放。

## 2026-04-13 当前排查结论

### 已确认的事实

1. 正式站首页广告容器已经渲染成功
2. 当前首页广告位 `slot id` 为 `6517724385`
3. VPS 可以正常下载：

```bash
curl -L --output /tmp/adsbygoogle.js "https://pagead2.googlesyndication.com/pagead/js/adsbygoogle.js?client=ca-pub-7143421934912272"
```

4. 当前本地异常网络环境里，Google 广告域名会被解析到 `198.18.x.x`
5. 在该异常环境下，浏览器会报：
   - `net::ERR_CONNECTION_CLOSED`

### 当前结论

- 不是 `slot id` 配置错误
- 不是前端广告组件没有渲染
- 不是 VPS 无法访问 Google
- 当前主要问题在于：部分终端访问环境无法稳定获取 Google 广告脚本

## 当前建议

1. 不要把当前异常网络环境作为唯一验收依据
2. 继续在正常网络环境里按本方案复核
3. 如果主要用户在中国大陆，不建议把 AdSense 当作唯一前台广告方案
4. 可保留 AdSense，同时准备自营广告 / CPS / 下载中间页推广位作为降级方案
